2. Authentication Flow

sequenceDiagram participant C as Client participant F as JwtAuthFilter participant AC as AuthController participant AS as AuthServiceImpl participant UDS as UserDetailsServiceImpl participant DB as PostgreSQL C->>AC: POST /auth/login {email, password} AC->>AS: login(request) AS->>UDS: loadUserByUsername(email) UDS->>DB: SELECT user WHERE email & is_active DB-->>UDS: User entity UDS-->>AS: UserDetails AS->>AS: Verify password (BCrypt) AS->>DB: INSERT refresh_token (SHA-256 + pepper) AS-->>C: {accessToken (15min), refreshToken (7d)} Note over C,F: Subsequent requests C->>F: Authorization: Bearer {accessToken} F->>F: Decode JWT, set SecurityContext F-->>C: 401 if invalid/expired C->>AC: POST /auth/refresh {refreshToken} AC->>AS: refresh(token) AS->>DB: Revoke old token, issue new pair AS-->>C: {new accessToken, new refreshToken}
